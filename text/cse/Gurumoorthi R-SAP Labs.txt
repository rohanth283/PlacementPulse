empty parts such that in at least one part the letter x and the letter y occur same number of times. 
Return number of splits. 
 
Example 1: 
s = “ayxbx” 
Output: 3 
 
 
 
Explanation: 
Split 1: a/yxbx -> first part “a” has equal occurrence of x and y (x = 0, y = 0) 
Split 2: ayx/bx -> first part “ayx” has equal occurrence of x and y (x = 1, y = 1) 
Split 3: ayxb/x -> first part “ayxb” has equal occurrence of x and y (x = 1, y = 1) 
No other valid split is there. 
So, output is 3.

Question 1: Searching order with Ranking 
Problem Statement: 
• 
A set of words is given, each with a rank. 
• 
When a user types in the search bar, all words matching the prefix of the search key should 
be displayed. 
• 
The results should be ordered by rank (descending). 
Example: 
Words with rank: 
app (10), application (8), apple (9), apply (5), banana (15) 
 
Search Key = "app" 
Result = app (10) → apple (9) → application (8) → apply (5) 
 
(Note: “banana” is ignored since it does not match the prefix "app") 
My Thought Process & Approach: 
• 
Since the problem deals with prefix matching, the first idea that came to my mind was to use 
a Trie data structure. 
o 
Each word can be inserted into the Trie. 
o 
Searching for the prefix is efficient (O(length of prefix)). 
• 
After finding the prefix node in the Trie, we need to return all words under that prefix sorted 
by rank. 
Clarification I Asked: 
• 
I asked the interviewer: “If there is a word like banana(15), should it appear when the search 
key = app?” 
• 
The interviewer said No, since only prefix matches are valid. 
• 
That confirmed my Trie-based approach was correct. 
Optimization Discussion: 
• 
My initial idea: Insert all words in the Trie → When searching, collect words under the prefix 
→ Sort them by rank before displaying. 
• 
Interviewer expectation: Further optimize. 
o 
Sorting every time can be costly if there are many words. 
o 
Possible optimization: At each Trie node, maintain a Heap (Max-Heap or Min-Heap) 
or a pre-sorted list of top words for quick retrieval. 
• 
We discussed how using a Heap along with Trie could give better performance for large 
datasets. 
 
 
Question 2: Infection Spread in Cities (Grid + BFS) 
Problem Statement: 
• 
A grid is given where each cell represents a city. 
• 
Each cell has two possible states: 
o 
1 → City is already infected 
o 
0 → City is not infected 
• 
Infected cities spread the infection to their adjacent (up, down, left, right) cities in 1 day. 
• 
The task is to determine the day on which all the cities become infected. 
 
 
My Approach & Explanation: 
• 
I quickly recognized this as a variation of the  
Rotten Oranges problem(https://leetcode.com/problems/rotting-oranges/) 
• 
The natural approach here is to use Breadth-First Search (BFS): 
o 
Treat all initially infected cities (1) as sources and push them into a queue with day = 
0. 
o 
Perform BFS to spread infection level by level (day by day). 
o 
Track the maximum day reached when the last city becomes infected. 
Code Writing: 
• 
The interviewer asked me to write the code. 
• 
I implemented the BFS solution step by step: 
1. Traverse grid → push all 1 cells into queue with day = 0. 
2. BFS → spread to adjacent cells and mark them infected. 
3. Keep track of days. 
4. Return the maximum day. 
Edge Cases (Discussed with Interviewer): 
1. No city is infected initially → Answer should be 0 (no spread possible). 
2. All cities are already infected → Answer should be 0 (no extra days needed). 
3. Some cities can never be infected (isolated regions with no adjacent infected cities) → 
Should return -1 or handle accordingly. 
 
 
Question 3: Designing a Stack with Min/Max Operations 
Problem Statement: 
• 
Design a Stack that supports the following operations: 
o 
push(x) → Add element at the top of the stack. 
o 
pop() → Remove the element at the top of the stack. 
o 
getMin() → Return the minimum element present in the stack. 
o 
getMax() → Return the maximum element present in the stack. 
• 
The stack should efficiently handle MinElement and MaxElement queries. 
 
 
 
My Approach & Explanation: 
1. Brute Force Approach (Initial Idea): 
o 
Use the standard stack container in C++ for push and pop. 
o 
For getMin() and getMax(), traverse the entire stack to find min/max. 
o 
Simple but inefficient → O(n) time for min/max queries. 
2. Optimized Approach (Inspired by MinStack Problem): 
o 
Store each element as a pair (a, b), where: 
▪ 
a = element itself 
▪ 
b = minimum element in the stack till now 
o 
Similarly, maintain another stack (or extend pair) to track maximum element till 
now. 
o 
This way, push, pop, getMin(), and getMax() all work in O(1) time. 
Discussion with Interviewer: 
• 
I explained both brute force and optimized approaches. 
• 
We discussed pros and cons: 
o 
Brute force → Easy to implement but slow for large inputs. 
o 
Optimized (with pairs) → Extra space usage but very efficient. 
• 
The interviewer appreciated that I explained the trade-offs and thought about scalability.

Tips & Notes: 
For Coding Round: 
• 
Confidence is key → Don’t panic if you get stuck, break the problem into smaller parts. 
• 
Attempt all questions → Even partial code can fetch marks, so don’t leave questions blank. 
• 
Start with Brute Force → First write a correct solution, then optimize step by step. 
• 
Think about Edge Cases → Empty input, single element, large constraints, negative values, etc. 
• 
Practice previous year questions → Get an idea of the difficulty level and common patterns. 
 
• 
Time Management → Don’t spend too long on one problem, move to the next and come back 
later. 
• 
Dry run your code → Before submitting, test with small cases manually. 
• 
Stay calm during OA → Focus more on accuracy first, then optimization. 
 
 
 
For Interviews: 
• 
Be confident – Confidence is the most important quality throughout the interview process. 
• 
Revise your resume – Before every round, go through what you have mentioned in your 
resume. You should be able to explain every point with clarity. 
• 
Research the company – Be curious about the company and the role you are applying for. This 
shows your genuine interest. 
• 
Show eagerness to learn – Even if you don’t know everything, show that you are a quick and 
eager learner. 
• 
Have a good discussion – Don’t just answer; interact with the interviewer. Try to connect with 
them by asking meaningful questions about their work and experience. 
• 
Think before answering – When a question is asked, take a moment to think. Then explain 
your answer clearly and confidently. 
• 
Handle unknown questions smartly – Even if you don’t know the exact answer, try to say 
something related to the question and show your thought process. Interviewers often value 
the way you think.
