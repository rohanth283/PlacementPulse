1.ONLINE TEST: 
The first stage was a pure coding round conducted on the HackerRank platform, with 
a total duration of 3 hours. The test consisted of 4 programming questions 
• 1 Easy 
• 
2 Moderate 
• 
1 Hard 
The test allowed candidates to code in any programming language of their choice. 
 
1. Developer Team Formation: Each developer has skill levels from 1 to n, and 
limits on how many teammates with lower or higher skills they can work with. 
A developer is “content” if these limits are satisfied. The task was to form the 
largest possible team where everyone is content. 
Example: n=3, lowerSkill=[0,1,2],  
higherSkill=[2,1,0] → Team {1,2,3} 
    works because all developers’ limits are satisfied → largest team size = 3. 
 
 
2. Largest Subset of Events: In a scheduling system, each event has a start and 
end time. A subset of events is called high-priority if every event overlaps or 
touches every other event in the subset. The task was to find the maximum-
size high-priority subset. 
Example: start=[1,2,3,4] 
     finish=[2,3,5,6] → Subset {2,3,4} 
     overlaps fully → largest subset size = 3. 
 
 
3. Minimum Node Types in a Graph: Given an undirected graph, the goal was to 
assign a type to each node so that: 
o Connected nodes have different types 
o Nodes sharing a neighbor also have different types 
The objective was to minimize the number of types used. 
       Example: n=4 
 edges=[(1,2),(2,3),(2,4),(3,4)] → Minimum types required = 3. 
 
 
4. Server State Flipping: A binary string represented server states (‘1’ = active, 
‘0’ = inactive), and a list of operations defined ranges to flip. The task was to 
find how many distinct active server counts can be obtained by applying any 
subset of operations. 
Example: servers="101",  
operations=[(0,1),(1,2)] → Possible active counts = {1,2}.

3.TECHNICAL and HR INTERVIEW (30-40 Minutes): 
Within just 10 minutes, I was called for the next round. This one started chill, again 
with the usual self-introduction and project discussion. 
This round had some comparatively tougher queries than the previous one. I was 
asked to explain the difference between RANK() and DENSE_RANK(), followed by a 
few more advanced SQL queries.  
One of the questions was how to convert a row-oriented table into a column-oriented 
format, for which I explained the approach using window functions. 
Next, I was given a coding problem: an expense splitter/tracker. The problem was: 
• 
A contributes 1200, B contributes 800, C contributes 1000 for a trip. 
• 
After calculating each person’s fair share, I was asked to determine who 
should give money to whom and how much. 
• 
For this example: A had paid 200 extra, so the system should return 200 to A 
from B, while C’s contribution remained the same. 
 
After solving this , a few reasoning and math questions were asked. For instance: 
• If you start with 100 Rs, gain 10% profit at the end of the  day, and 10% loss 
the next day,what is the final loss?
