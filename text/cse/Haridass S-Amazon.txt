1. Problem Solving 
Question 1 – Dataset Cleaning Cost 
You are given a string dataset of even length consisting of lowercase English letters. You can remove any two 
characters in one operation: 
• 
If both are the same → cost = matchCost 
• 
If different → cost = mismatchCost 
Your goal is to remove all characters while minimizing the total cost. 
Example: 
• 
dataset = "ouio" 
• 
matchCost = 2 
• 
mismatchCost = 4 
Step-by-step Explanation: 
1. The string is "ouio" → characters: 'o', 'o', 'u', 'i'. 
2. Remove 'o' and 'o' → cost = 2 → remaining string = "ui". 
3. Remove 'u' and 'i' → cost = 4. 
Total Cost = 2 + 4 = 6  
 
Question 2 – System Stabilization 
You are given a string pipeline representing service nodes. A node type failedService disrupts the system: 
• 
Every second, each occurrence of failedService removes the preceding character. 
• 
All removals happen simultaneously every second. 
• 
Continue until no more removals occur. Return the number of seconds for stabilization. 
 
Example: 
• 
pipeline = "database" 
• 
failedService = 'a' 
Step-by-step Explanation: 
• 
Initial pipeline: "database" 
• 
Time = 1 → Each 'a' removes preceding characters → removes 'd' and 't' → remaining string = "aaase" 
• 
Time = 2 → Each 'a' removes preceding characters → remaining string = "ase" 
Output = 2 seconds

The interview began with a brief introduction, followed by two live coding problems emphasizing optimal

Question 1 – Word Break 
Determine if a string s can be segmented into words from wordDict (reuse allowed). Optimal DP/memoization 
solutions expected. 
• 
Example: 
o 
s = "applepenapple" 
o 
wordDict = ["apple", "pen"] 
o 
Output: true 
o 
Explanation: "applepenapple" → "apple pen apple" 
Question 2 – Sum of Prefix Scores of Strings 
Given an array of words, the score of a string is the count of words that have it as a prefix. Efficient Trie-based 
logic expected. 
• 
Example: 
o 
words = ["abc", "ab", "bc", "b"] 
o 
Output = [5, 4, 3, 2] 
o 
Explanation: 
▪ 
"abc" → prefixes: a, ab, abc → counts: 2 + 2 + 1 = 5 
▪ 
"ab" → a, ab → 2 + 2 = 4 
▪ 
"bc" → b, bc → 2 + 1 = 3 
▪ 
"b" → b → 2 
Bonus Discussion – MST Edge Classification 
Discussion on Critical and Pseudo-Critical Edges in MST, Kruskal’s Algorithm, Union-Find, and identifying edges 
that increase MST weight when removed.
