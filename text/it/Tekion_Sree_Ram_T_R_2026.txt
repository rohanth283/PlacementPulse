DSA Question Asked

Problem Statement: You are given an integer array arr[] of size n and an integer k. Your task is
to find the minimum length subarray of arr that contains all k consecutive integers (numbers
forming a continuous range of length k).

The subarray does not need to have elements in sorted order. If no such subarray exists, return
-1.

Example:

Input: arr = [7, 3, 5, 6, 4, 9]
k = 3

Explanation:

- Subarray [3,5,6,4] → contains {3,4,5,6} → includes {3,4,5} → length = 4
- Subarray [5,6,4] → contains {4,5,6} → ✅ valid, length = 3 (minimum so far)
- Subarray [7,3,5,6,4] → contains {3,4,5,6,7} → length = 5

Minimum length = 3

2. Interview at CUIC

______________________________________________________________________________

i) Offline Coding Round 1

Frontend Section: - Task: Create a dynamic form builder using HTML, CSS, and JavaScript.
- Focus: DOM manipulation, event handling.

Backend Section: - Two coding problems:

1. Buy and Sell Stock II (LeetCode): Find maximum profit with unlimited transactions

using a greedy approach.

2. User Activity Log Analysis: Two files (Day 1 and Day 2) with timestamped user visits.
Retrieve users visiting two different pages on both days. Initially solved with Python file
handling, then re-implemented using CSV format.

______________________________________________________________________________

ii) Offline Coding Round 2 (DSA Focused)

1. Word Break II (LeetCode) - Input: string s and wordDict. Return all sentences by inserting
spaces so every word exists in wordDict. Words can be reused. - Example: s = "catsanddog",
wordDict = ["cat","cats","and","sand","dog"] → Output: ["cats and dog","cat sand dog"] - Solved using
recursion with backtracking.

2. Longest String with Limited Character Repetition - Input: integers a, b, c for counts of ‘a’, ‘b’,
‘c’. Construct the longest string without three consecutive identical letters. - Example: a = 1, b =
1, c = 7 → Output: ccbccacc - Solved using a priority queue (max-heap) strategy.

5. Puzzle:

You have two ropes of different lengths. Each rope takes exactly 1 hour to burn, but the
burning rate is not uniform. You need to find the middle point of one rope using the
other.
I was unable to solve it initially. The interviewer explained:

· Burn one rope from both ends. When it finishes, exactly half of the other rope

has burned.

· He related this concept to slow and fast pointer technique in finding the middle

of a linked list.

3. Key Suggestions and Tips from My Experience

1. Be strong in DSA and problem-solving: Arrays, strings, recursion, backtracking, priority

queues, tree traversals.

2. Be clear about everything in your resume: Skills, projects, tech stack, challenges,

unique features.

3. Prepare separate self-intros for technical and HR rounds: Technical intro – projects and

skills; HR intro – academics, hobbies, extracurriculars.

4. Be confident: Explain your approach logically.
