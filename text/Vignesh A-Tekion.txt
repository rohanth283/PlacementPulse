DSA: 
Problem 1: 
You are given two arrays, marie and annie, which represent the face cards of two players, and a 
string flip that contains either "Even" or "Odd". The game is played in rounds, where each index i 
of the arrays represents one round. 
• 
If flip = "Even", then the players calculate their scores as: 
o Marie’s score for this round = abs(marie[i] – annie[i]) 
o Annie’s score for this round = abs(annie[i] – marie[i]) 
• 
If flip = "Odd", both players skip their cards for that round (no score is added). 
After each round, the value of flip toggles (i.e., "Even" becomes "Odd", and "Odd" becomes 
"Even"). 
Your task is to find the final score of Marie and Annie after all rounds. 
 
Example: 
Input: 
marie = [4, 5, 6, 7]   
annie = [1, 2, 3, 4]   
flip = "Even" 
Process: 
• 
Round 1 (flip = "Even"): 
Marie = |4 – 1| = 3 
Annie = |1 – 4| = 3 
• 
Round 2 (flip = "Odd"): skipped 
• 
Round 3 (flip = "Even"): 
Marie = |6 – 3| = 3 
Annie = |3 – 6| = 3 
• 
Round 4 (flip = "Odd"): skipped 
Final Scores: 
Marie = 3 + 3 = 6   
Annie = 3 + 3 = 6 
 
Problem 2: Bandwidth Allocation (0/1 Knapsack Variant) 
You are given: 
• 
An array bandwidthNeeded[ ], where bandwidthNeeded[i] represents the amount of 
bandwidth required to process the i-th process. 
• 
An array requestsProcessed[ ], where requestsProcessed[i] represents the number of 
requests that can be processed if the i-th process is allocated its required bandwidth. 
• 
An integer totalBandwidth, which represents the maximum total bandwidth available. 
You need to allocate the available bandwidth to processes in such a way that: 
• 
A process is either fully allocated its required bandwidth or not allocated at all (no partial 
allocation). 
• 
The total allocated bandwidth does not exceed totalBandwidth. 
• 
The total number of requests processed is maximized. 
Your task is to determine the maximum number of requests that can be processed with the given 
total bandwidth. 
Example 
Input: 
bandwidthNeeded = [2, 3, 4, 5]   
requestsProcessed = [3, 4, 5, 6]   
totalBandwidth = 5 
Explanation: 
• 
If we choose processes 1 and 2: bandwidth used = 2 + 3 = 5 → requests processed = 3 + 4 
= 7 
• 
If we choose processes 3: bandwidth used = 4 → requests processed = 5 
• 
If we choose process 4: bandwidth used = 5 → requests processed = 6 
The optimal allocation is processes 1 and 2 → 7 requests. 
Output: 
Maximum requests processed = 7

several DBMS-related questions.

3. What is a transaction? 
4. What are the ACID properties? 
5. What are the different types of joins? 
6. What is functional dependency? 
7. What is a read/write conflict?

DSA Problems 
He then moved on to DSA and asked me three problems: 
1. Given an array and integer k, left shift the array by k times. 
2. Find the parent node of a given node in a binary tree. 
3. Find the intersection node of two linked lists. 
I provided both brute-force and optimal approaches for all the problems and wrote the pseudo-

Key Notes: 
• 
Revise your core subjects like DBMS, OS, Computer Networks, and DSA thoroughly before 
the interview. 
• 
Practice solving coding problems and be prepared to explain both brute force and optimal 
approaches. 
• 
Always mention the time and space complexity when explaining your solutions. 
• 
Be confident in your projects and clearly explain the technologies used, design choices, and 
outcomes. 
• 
Communicate your answers in a structured and confident manner. 
• 
Prepare for behavioral questions such as teamwork, conflict management, strengths, and 
weaknesses. 
• 
Stay calm during the interview and admit honestly if you don’t know something, while trying 
to reason it out. 
• 
Show genuine interest in the company by asking thoughtful questions at the end of the 
interview.
