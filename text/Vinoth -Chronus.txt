My piece of advice: 
 
● Practice regularly on leetcode and hackerrank or even gfg. 
● Complete 
https://www.geeksforgeeks.org/must-do-coding-questions-for-companies-like-
amazon-microsoft-adobe/ 
●  
● Concentrate on OS and DBMS too.(You will have to know the code for thread creation in a 
language of your choice and code for JDBC connectivity. Boomer interviewers ask stupid 
questions like these) 
● Develop an intuition for recursion 
● Dynamic programming might be a bit counterintuitive but you will love it once you get a 
knack of it. Start with a recursive solution ,then formulate top down dp approach then 
proceed to bottom up dp.(My way of doing dp. Not sure if it's perfect). 
● Have 2-3 solid projects (Atleast) 
● Do not over-familiarize yourself with standard problems .Try competitive programming .Try 
doing long challenges on codechef.
